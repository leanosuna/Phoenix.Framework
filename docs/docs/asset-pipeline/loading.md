# Loading Assets

Assets are loaded at runtime from a binary content folder resolved via a JSON manifest.

## AssetLoader

Static class for loading models, textures, and shaders.

### Initialization

`AssetLoader.Init()` is called automatically by `PhoenixGame.DelayedLoad()`. You can customize paths:

```csharp
// Default:
AssetLoader.Init(this);  // Uses Content/ and Content/asset-manifest.json

// Custom paths:
AssetLoader.Init(this,
    contentPath: "Data/assets/",
    manifestPath: "Data/assets/asset-manifest.json"
);
```

### Loading Models

```csharp
// Load by relative path (without extension)
var model = AssetLoader.LoadModel("characters/robot");

// Cast to AnimatedModel if needed
var animatedModel = (AnimatedModel)AssetLoader.LoadModel("characters/robot");

// Load static model
var staticModel = (Model)AssetLoader.LoadModel("props/wooden-box");
```

### Loading Textures

```csharp
var texture = AssetLoader.LoadTexture("textures/brick-albedo");
```

### Loading Shaders

```csharp
var shader = AssetLoader.LoadShader("shaders/pbr");
// Looks for "shaders/pbr.vert" and "shaders/pbr.frag"
```

## Asset Manifest

A JSON file that maps asset names to file paths.

### Structure

```json
{
    "BaseDirectory": "C:/projects/MyGame/",
    "Assets": [
        {
            "Path": "C:/projects/MyGame/ContentBin/characters/robot.bin",
            "RelativePath": "characters/robot",
            "Type": 0
        },
        {
            "Path": "C:/projects/MyGame/ContentBin/textures/brick-albedo.bin",
            "RelativePath": "textures/brick-albedo",
            "Type": 1
        },
        {
            "Path": "C:/projects/MyGame/ContentBin/shaders/pbr.vert",
            "RelativePath": "shaders/pbr",
            "Type": 2
        },
        {
            "Path": "C:/projects/MyGame/ContentBin/shaders/pbr.frag",
            "RelativePath": "shaders/pbr",
            "Type": 2
        }
    ]
}
```

### AssetType Enum

| Value | Name | Description |
|-------|------|-------------|
| 0 | `Model` | Binary model file (`.bin`) |
| 1 | `Texture` | Binary texture file (`.bin`) |
| 2 | `Shader` | Shader stage file (`.vert` / `.frag`) |
| 3 | `Unknown` | Unrecognized asset type |

### Loading by AssetEntry

```csharp
var manifest = JsonIOTools.Load<AssetManifest>("path/to/manifest.json");
// manifest.BaseDirectory  — base path for assets
// manifest.Assets         — list of AssetEntry
```

## Asset Resolution Process

When you call `AssetLoader.LoadModel("characters/robot")`:

```
1. Remove extension: "characters/robot" → "characters/robot"
2. Search manifest for matching RelativePath
3. Get absolute path: BaseDirectory + ContentBinBaseDirectory + RelativePath
4. Load via BinaryModelReader
5. Cache in _loadedModels dictionary
6. Return cached instance on subsequent calls
```

## Caching

All loaded assets are cached in static dictionaries:

```csharp
_loadedModels:   Dictionary<string, Model>
_loadedTextures: Dictionary<string, GLTexture>
_loadedShaders:  Dictionary<string, GLShader>
```

Subsequent calls with the same name return the cached instance:

```csharp
var model1 = AssetLoader.LoadModel("characters/robot");
var model2 = AssetLoader.LoadModel("characters/robot");
// model1 == model2  (same instance)
```

## Content Folder Structure

```
YourGame/
 ├── ContentBin/               # Compiled binary assets
 │   ├── characters/
 │   │   └── robot.bin
 │   ├── props/
 │   │   └── wooden-box.bin
 │   ├── textures/
 │   │   ├── brick-albedo.bin
 │   │   └── brick-normal.bin
 │   └── shaders/
 │       ├── pbr.vert
 │       └── pbr.frag
 ├── Content/                  # Source assets (original files)
 │   ├── characters/
 │   │   └── robot.fbx
 │   └── textures/
 │       └── brick.png
 └── asset-manifest.json       # Generated by AssetTool
```

## Source Files vs Binary Files

| Type | Source Location | Binary Location | Format |
|------|----------------|-----------------|--------|
| Models | `Content/` (FBX, OBJ, etc.) | `ContentBin/` (`.bin`) | Custom binary |
| Textures | `Content/` (PNG, JPG, etc.) | `ContentBin/` (`.bin`) | BC1/BC3/BC5 compressed |
| Shaders | `Content/` (`.vert`, `.frag`) | `ContentBin/` (`.vert`, `.frag`) | Plain GLSL text |

Shaders are loaded directly as text files (not compiled to binary).

## Error Handling

If an asset is not found in the manifest, `LoadModel()`, `LoadTexture()`, and `LoadShader()` will throw an exception. If a shader has no matching `.vert`/`.frag` pair, it throws with a descriptive error message.
